---
title: "ko.identify()"
description: "Identify visitors and associate events with known users"
---

<Info>
  **Purpose**: Associate a visitor with an email address and enrich their profile with traits
</Info>

## Syntax

```js
ko.identify(email, traits, options)
```

## Parameters

<ParamField path="email" type="string" required>
  The visitor's email address. This is used to associate the visitor with a company and track them across devices.
</ParamField>

<ParamField path="traits" type="object">
  Additional properties about the visitor or their account.
  
  <Expandable title="Visitor Traits">
    <ParamField path="name" type="string">
      The visitor's full name
    </ParamField>
    <ParamField path="title" type="string">
      The visitor's job title
    </ParamField>
    <ParamField path="role" type="string">
      The visitor's role or department
    </ParamField>
  </Expandable>
  
  <Expandable title="Account Traits">
    <ParamField path="$account" type="object">
      Company or organization-level data
      
      <ParamField path="$account.group_id" type="string">
        Unique identifier for the account (recommended)
      </ParamField>
      <ParamField path="$account.name" type="string">
        Company name
      </ParamField>
      <ParamField path="$account.domain" type="string">
        Company domain
      </ParamField>
      <ParamField path="$account.plan" type="string">
        Subscription plan or tier
      </ParamField>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="options" type="object">
  Additional configuration options (optional)
</ParamField>

## Code Examples

### Basic Identification

Identify a visitor with just their email address:

```js
ko.identify('jane.doe@example.com')
```

### With Visitor Traits

Include additional information about the visitor:

```js
ko.identify('jane.doe@example.com', {
  name: 'Jane Doe',
  title: 'Product Manager',
  role: 'Product'
})
```

### With Account Traits

Associate the visitor with account-level data:

```js
ko.identify('jane.doe@example.com', {
  name: 'Jane Doe',
  title: 'Product Manager',
  $account: {
    group_id: 'acme-production-id',
    name: 'Acme Inc.',
    domain: 'example.com',
    plan: 'Enterprise'
  }
})
```

### Multiple Account Groups

Link the same user to multiple account groups:

```js
// Production environment
ko.identify('jane.doe@example.com', {
  $account: {
    group_id: 'acme-production-id',
    name: 'Acme Inc.',
    domain: 'example.com',
    plan: 'Enterprise'
  }
})

// Test environment
ko.identify('jane.doe@example.com', {
  $account: {
    group_id: 'acme-test-environment-id',
    name: 'Acme Inc.',
    domain: 'example.com',
    plan: 'Free'
  }
})
```

## When to Call

You should call `ko.identify()` whenever a visitor:

- **Signs up** for your product for the first time
- **Logs in** to your application
- **Provides their email** in a form
- **Loads a page** as a logged-in user

## Special Use Cases

### Google Sign In & SSO

For Google Sign-In, GitHub Sign-In, or SSO, call `ko.identify()` after successful login:

```js
// After successful OAuth/SSO login
ko.identify('jane.doe@example.com', {
  name: 'Jane Doe',
  title: 'Product Manager'
})
```

<Note>
  Install the Koala pixel on your logged-in site's landing page and call `ko.identify()` from there. For server-side identification, see [Server-side Implementation](/developer-guides/server-side#identifying-visitors).
</Note>

### Anonymous Visitors

You can identify anonymous visitors with traits before knowing their email:

```js
ko.identify(null, {
  source: 'organic_search',
  campaign: 'product_launch'
})
```

## Best Practices

<Tip>
  **Always include traits**: Send visitor and account traits with every identify call to keep profiles up-to-date.
</Tip>

<Tip>
  **Use group_id**: Always include a unique `group_id` for account traits to avoid conflicts.
</Tip>

<Warning>
  **Call ko.reset() on logout**: Prevent future events from being linked to the logged-out user.
</Warning>
